## Dev Notes

> **CRITICAL:** Toda informação técnica abaixo veio de `docs/architecture.md` e `docs/prd.md` (citados). Nada inventado.

### Route Handler no App Router

Route Handlers vivem em `app/health/route.ts` e exportam funções HTTP (`GET`, `POST`, etc.). São server-side por padrão — sem `'use server'` necessário. [Source: architecture.md § 10.3 — listado como `app/health/route.ts`]

`export const dynamic = 'force-dynamic'` desabilita caching estático do Next.js — crítico para health check sempre retornar dados frescos.

### Supabase Client no Route Handler

Usar `createBrowserClient` (sem cookies) é suficiente para uma query de health check — não há autenticação necessária. Para handlers que precisam de cookies (auth), usar `createServerClient`. [Source: architecture.md § 11.2]

```typescript
import { createBrowserClient } from '@supabase/ssr';

const supabase = createBrowserClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
);
```

### Resposta JSON Esperada

```json
{
  "status": "ok",
  "db": { "reachable": true, "latencyMs": 45 },
  "commit": "abc123def",
  "env": "production"
}
```

- `VERCEL_GIT_COMMIT_SHA`: variável injetada automaticamente pela Vercel (short SHA do commit).
- `VERCEL_ENV`: `"production"` | `"preview"` | `"development"` — injetado pela Vercel.
- Em desenvolvimento local: `commit = "local"`, `env = "development"`.

### Estratégia de Query de Conectividade

Options (em ordem de preferência):
1. `supabase.auth.getSession()` — não requer tabela, apenas conectividade de rede.
2. `supabase.rpc('now')` — requer `create function public.now()` (overhead de migration).
3. `supabase.from('profiles').select('id').limit(1)` — funciona após Story 2.1 (profiles table); premature dependency nesta story.

**Recomendação:** usar `auth.getSession()` que é sempre disponível via Supabase Auth.

### Teste Unitário — Mock Strategy

```typescript
import { vi } from 'vitest';

vi.mock('@supabase/ssr', () => ({
  createBrowserClient: vi.fn(),
}));
```

### Encoding Patterns

- **Absolute Imports:** `@/lib/...` via tsconfig alias. [Source: architecture.md § 16]
- **Conventional Commits:** `feat: add canary health route [Story 1.7]`. [Source: architecture.md § 16]
