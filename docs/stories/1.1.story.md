## Story

**Como** dev mantenedor do youtube-biolink,
**eu quero** inicializar o app Next.js 16 com TypeScript strict e Tailwind configurados,
**para que** eu tenha uma base sólida, tipada e estilizada para todas as próximas stories do MVP.

## Acceptance Criteria

> Fonte canônica: `docs/prd.md` § 6 — Epic 1, Story 1.1.

1. **Scaffold via `create-next-app@latest`** executado com flags equivalentes a: App Router, TypeScript, Tailwind, ESLint, **sem** `src/`, import alias `@/*`, usando Next.js **16.x**. O `package.json` resultante registra `"next": "^16.x"` e `"react": "^19.x"`.
2. **`tsconfig.json`** com `"strict": true`, `"noImplicitAny": true`, `"strictNullChecks": true` ativos explicitamente (mesmo que herdados de `strict: true`, devem estar visíveis no arquivo) e `"paths": { "@/*": ["./*"] }` configurado.
3. **`pnpm dev`** inicia o servidor em `http://localhost:3000` e renderiza a página inicial gerada pelo scaffold sem erros no terminal nem warnings de hidratação no console do browser.
4. **`pnpm build`** roda até o fim sem erros e produz build de produção válida (`.next/` populada).
5. **Commit inicial** no padrão conventional commits: `feat: initialize next.js 16 project [Story 1.1]` — assinado conforme `commitlint` (commitlint só será enforced na Story 1.2; aqui o formato é manual mas obrigatório).

### Coding Standards aplicáveis nesta story

- **Absolute Imports:** alias `@/*` configurado em `tsconfig.json` aponta para `./*` (raiz do app, **sem** `src/`). Não há código de domínio nesta story, mas a configuração precisa estar correta para stories subsequentes. [Source: architecture/architecture.md § 16]
- **Conventional Commits:** `feat(scope?): descrição [Story X.Y]` — nesta story, o scope pode ser omitido (commit de bootstrap). [Source: architecture/architecture.md § 16, PRD NFR9]
- **TypeScript strict:** zero `any` implícito permitido. [Source: PRD NFR5]

### Pontos críticos / armadilhas

1. **`src/` existente vazio:** o diretório `src/` na raiz é resíduo do bootstrap AIOX (vazio). Precisa ser removido **antes** de rodar `create-next-app` com `--no-src-dir`, senão o tool pode falhar ou ser inconsistente.
2. **`tests/` existente vazio:** mantém-se para a Story 1.3 (Vitest); não precisa ser removido.
3. **`AGENTS.md` na raiz:** não conflita com scaffold; preservar.
4. **`.env` versus `.env.local`:** o `.env` atual contém credenciais de bootstrap AIOX. `create-next-app` não toca em `.env*`. A migração para `.env.local` (PRD NFR10) acontece na Story 1.4 quando as credenciais Supabase entrarem. **Nesta story, não criar `.env.local`**.
5. **Tailwind 4 vs Tailwind 3:** Next 16 com `create-next-app@latest` gera Tailwind 4 (sintaxe `@import "tailwindcss"` em CSS, sem `tailwind.config.ts` tradicional — config via CSS `@theme`). Isso é **esperado** e alinhado com arquitetura § 3.1.
6. **commitlint:** ainda não está instalado (Story 1.2). O commit desta story segue o padrão **manualmente**.
7. **Husky pre-commit hooks:** idem — Story 1.2.
8. **Branch:** working branch sugerida `feature/1.1-bootstrap-nextjs` (criada localmente pelo `@dev`/`@sm` antes do scaffold). Branch protection da `main` só será ativada na Story 1.5.
