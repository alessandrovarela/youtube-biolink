## Story

**Como** aprendiz clonando o repositório,
**eu quero** um README com instruções de setup local completas e um diretório `docs/stories/` organizado,

    ```markdown
    # youtube-biolink

    ![CI](https://github.com/alessandrovarela/youtube-biolink/actions/workflows/ci.yml/badge.svg)

    > Projeto didático fullstack: Next.js 16 + Supabase + Vercel.
    > Aprenda autenticação, banco de dados, SSR, CI/CD, analytics e design system
    > construindo um clone simplificado do Linktree do zero.

    ## Stack

    Next.js 16 · TypeScript strict · Tailwind 4 · Supabase Cloud · Vercel · pnpm 9

    ## Pré-requisitos

    - Node.js 20+
    - pnpm 9 (`npm install -g pnpm`)
    - Conta gratuita no [Supabase](https://supabase.com) (ou solicitar credenciais do projeto `development` ao owner)

    ## Setup local

    1. Clone o repositório: `git clone https://github.com/alessandrovarela/youtube-biolink`
    2. Entre no diretório: `cd youtube-biolink`
    3. Copie o arquivo de exemplo: `cp .env.example .env.local`
    4. Preencha as credenciais do projeto Supabase `development` em `.env.local`
    5. Instale as dependências: `pnpm install`
    6. Inicie o servidor: `pnpm dev`
    7. Acesse: http://localhost:3000

    > **Nota:** o projeto não requer Supabase local. Use o projeto `development` na nuvem.

    ## Comandos principais

    | Comando | Descrição |
    |---------|-----------|
    | `pnpm dev` | Servidor de desenvolvimento |
    | `pnpm build` | Build de produção |
    | `pnpm lint` | Verificar código |
    | `pnpm format` | Formatar código |
    | `pnpm test` | Executar testes |
    | `pnpm typecheck` | Verificar tipos TypeScript |

    ## Estrutura do projeto

    Ver [`docs/architecture.md`](docs/architecture.md) para arquitetura completa.
    Ver [`docs/stories/`](docs/stories/) para o material didático organizado por epic/story.

    ## Deploy

    Production: https://youtube-biolink.vercel.app
    ```
