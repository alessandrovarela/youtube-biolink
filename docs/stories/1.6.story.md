# Story 1.6 — Deploy automático na Vercel

## Status

## Story

**Como** dev mantenedor do youtube-biolink,
**eu quero** o projeto conectado à Vercel com deploy automático em `main` e previews em PRs,
**para que** todo merge gere uma URL pública e todo PR tenha preview para review.

## Acceptance Criteria

> Fonte canônica: `docs/prd.md` § 6 — Epic 1, Story 1.6.

1. Projeto importado na Vercel, vinculado ao repositório GitHub.
2. Variáveis de ambiente por escopo configuradas na Vercel: Production usa credenciais do projeto Supabase `production`; Preview e Development usam credenciais do projeto Supabase `development`.
3. Merge em `main` gera deploy em URL estável (ex.: `youtube-biolink.vercel.app`) conectado ao banco `production`.
4. PR gera preview URL automaticamente (conectada ao banco `development`), comentada pelo bot da Vercel.
5. Domínio de preview é acessível e serve a build da branch contra o banco `development`.

## Dev Notes

> **CRITICAL:** Toda informação técnica abaixo veio de `docs/architecture.md` e `docs/prd.md` (citados). Nada inventado.

### Estratégia de Deploy

- **Frontend + Backend:** Vercel (Next.js serverless). [Source: architecture.md § 13.1]
- **Environments:** Production → Supabase `production`. Preview + Development → Supabase `development`. [Source: architecture.md § 13.1]
- **Migrations:** `supabase db push` aplicado manualmente por devs no `development`; CI aplica em `production` após merge na `main`. [Source: architecture.md § 13.1]

### Variáveis de Ambiente por Scope

| Scope | `NEXT_PUBLIC_SUPABASE_URL` | `NEXT_PUBLIC_SUPABASE_ANON_KEY` |
|-------|----------------------------|----------------------------------|
| Production | URL do `youtube-biolink-prod` | anon key do `prod` |
| Preview | URL do `youtube-biolink-dev` | anon key do `dev` |
| Development | URL do `youtube-biolink-dev` | anon key do `dev` |

[Source: architecture.md § 2.2 — "Vercel: Env Vars por scope"]

### Sem arquivo de configuração Vercel

Não é necessário criar `vercel.json` para este setup básico — o framework Next.js é detectado automaticamente. Adicionar `vercel.json` apenas se customizações de headers/redirects forem necessárias (stories futuras).

### URL de Produção

Vercel gera automaticamente: `{repo-name}.vercel.app`. Pode-se configurar domínio customizado nas settings da Vercel (fora do escopo do MVP).
