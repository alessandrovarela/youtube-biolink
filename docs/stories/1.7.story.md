# Story 1.7 — Rota canary `/health`

## Status

**Ready** — Validado pelo `@po` (Pax). GO 8.5/10. Liberado para `@dev`. ⚠️ Should-Fix: substituir query `_health_check` por `auth.getSession()` em T1 (Dev Notes, Opção 1).

---

## Story

**Como** dev,
**eu quero** uma rota pública `/health` que retorne status da conexão com o Supabase,
**para que** eu tenha evidência ponta a ponta de que a cadeia app ↔ DB ↔ deploy funciona.

---

## Acceptance Criteria

> Fonte canônica: `docs/prd.md` § 6 — Epic 1, Story 1.7.

1. Rota `GET /health` implementada como Route Handler em `app/health/route.ts`.
2. Handler executa query trivial no Supabase (`select now()` via query direta) usando client server-side.
3. Resposta JSON: `{ "status": "ok" | "degraded", "db": { "reachable": boolean, "latencyMs": number }, "commit": "<sha>", "env": "production" | "development" }`.
4. Em caso de falha de conexão, retorna HTTP 503 com `status: "degraded"`.
5. Rota acessível publicamente na Vercel após deploy (em ambas as URLs: production e preview).
6. Teste unitário em `tests/unit/` cobre resposta em sucesso e em falha (mockando o Supabase client).

    ```typescript
    import { NextResponse } from 'next/server';
    import { createBrowserClient } from '@supabase/ssr';

    export const dynamic = 'force-dynamic'; // desabilita cache para health check

    export async function GET() {
      const start = Date.now();
      let reachable = false;

      try {
        const supabase = createBrowserClient(
          process.env.NEXT_PUBLIC_SUPABASE_URL!,
          process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
        );
        const { error } = await supabase.from('_health_check').select('now()').limit(1).maybeSingle();
        // Supabase retorna erro de "relation does not exist" mas a conexão funcionou
        // Alternativamente, usar RPC ou qualquer query que confirme conectividade
        reachable = true; // se chegou aqui sem network error, conexão ok
      } catch {
        reachable = false;
      }

      const latencyMs = Date.now() - start;
      const status = reachable ? 'ok' : 'degraded';
      const httpStatus = reachable ? 200 : 503;

      return NextResponse.json(
        {
          status,
          db: { reachable, latencyMs },
          commit: process.env.VERCEL_GIT_COMMIT_SHA ?? 'local',
          env: process.env.VERCEL_ENV === 'production' ? 'production' : 'development',
        },
        { status: httpStatus }
      );
    }
    ```

## Dev Notes

> **CRITICAL:** Toda informação técnica abaixo veio de `docs/architecture.md` e `docs/prd.md` (citados). Nada inventado.

### Route Handler no App Router

Route Handlers vivem em `app/health/route.ts` e exportam funções HTTP (`GET`, `POST`, etc.). São server-side por padrão — sem `'use server'` necessário. [Source: architecture.md § 10.3 — listado como `app/health/route.ts`]

`export const dynamic = 'force-dynamic'` desabilita caching estático do Next.js — crítico para health check sempre retornar dados frescos.

### Supabase Client no Route Handler

Usar `createBrowserClient` (sem cookies) é suficiente para uma query de health check — não há autenticação necessária. Para handlers que precisam de cookies (auth), usar `createServerClient`. [Source: architecture.md § 11.2]

```typescript
import { createBrowserClient } from '@supabase/ssr';

const supabase = createBrowserClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
);
```

### Resposta JSON Esperada

```json
{
  "status": "ok",
  "db": { "reachable": true, "latencyMs": 45 },
  "commit": "abc123def",
  "env": "production"
}
```

- `VERCEL_GIT_COMMIT_SHA`: variável injetada automaticamente pela Vercel (short SHA do commit).
- `VERCEL_ENV`: `"production"` | `"preview"` | `"development"` — injetado pela Vercel.
- Em desenvolvimento local: `commit = "local"`, `env = "development"`.

### Estratégia de Query de Conectividade

Options (em ordem de preferência):
1. `supabase.auth.getSession()` — não requer tabela, apenas conectividade de rede.
2. `supabase.rpc('now')` — requer `create function public.now()` (overhead de migration).
3. `supabase.from('profiles').select('id').limit(1)` — funciona após Story 2.1 (profiles table); premature dependency nesta story.

**Recomendação:** usar `auth.getSession()` que é sempre disponível via Supabase Auth.

### Teste Unitário — Mock Strategy

```typescript
import { vi } from 'vitest';

vi.mock('@supabase/ssr', () => ({
  createBrowserClient: vi.fn(),
}));
```

### Encoding Patterns

- **Absolute Imports:** `@/lib/...` via tsconfig alias. [Source: architecture.md § 16]
- **Conventional Commits:** `feat: add canary health route [Story 1.7]`. [Source: architecture.md § 16]
